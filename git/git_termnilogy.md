# repo - a git project tracked folder with a .git directory
# commit - a snapshot of your files and metadata
# branch - a movable pointer to a speciifc commit pointed toa branch (main,dev,feature)
# remote - a reference to an external git host usual named origin 
# staging area - this is called an index, a buffer between your working directory and the repo you plan to commit
# blobs - binary large objects, stores the atual contents of your files
# trees - represent a directory snapshot
# refs - pointer to a commit (branches,tags,special pointers)
# head - pointer to where you currently are
# index - binary file in disk that holds the staging area info
# object store - git stores all blobs, trees and commits by sha hash
# tag - ref to a specific commit often used to mark releases
